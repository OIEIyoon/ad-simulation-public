# Tasks 작성 가이드

## 목적

연구 태스크 및 진행 상황 문서화

## 구분

- **Research**: 연구 과제
- **Work**: 프로젝트/업무

## 구조

1. Objective - 목표 (1-2문장)
2. Overview - 전체 흐름 (mermaid)
3. 단계별 설명 - 표와 개조식으로
4. Status - 진행 상황
5. Related - 관련 노트 링크

## 스타일

- 열린 형태로 작성 (미정인 것은 "TBD", "미정")
- naive한 표현 지양 ("실패하면", "일" 등)
- 품위있는 연구 노트 어조
- mermaid flowchart로 파이프라인 시각화

## Front Matter

```yaml
---
title: 제목

---
```
